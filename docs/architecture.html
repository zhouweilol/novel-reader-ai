三、后端模块架构

NestJS 模块化设计，每个模块独立包含 Controller / Service / Entity

🔐 Auth 模块认证
auth.controller auth.service auth.module jwt.strategy
依赖：UsersModule · JwtModule · PassportModule
接口：POST /api/auth/register · POST /api/auth/login
安全：bcrypt 密码哈希(cost=10) · JWT 7天有效期 · 密钥: novel-reader-secret-key
📚 Books 模块图书
books.controller books.service book.entity
依赖：TypeORM Book Repository
接口：GET/POST /api/books · GET/PUT/DELETE /api/books/:id
权限：OptionalAuthGuard · findAll 区分 admin/owner/guest
📝 Chapters 模块章节
chapters.controller chapters.service chapter.entity doc-parser.utils
依赖：Book · IfStory · WritersModule
接口：GET books/:id/chapters · GET/POST/PUT/DELETE chapters · POST chapters/upload · POST chapters/batch
功能：MK/TXT文档解析 · 章节序号自动重排 · 500MB上传 · 每日配额检查
🔀 IF Stories 模块IF线
if-stories.controller if-stories.service if-story.entity
依赖：AiService · ChaptersService
接口：POST generate · POST submit · GET chapter/:id · POST tip/like · GET ranking · GET pending · PUT review
状态机：pending → approved / rejected
🤖 AI 模块AI配置
ai.controller ai.service ai-config.entity
依赖：ChaptersService
接口：GET/POST config · POST generate-if-story · POST chat · GET features
安全：apiKey AES-256-CBC 加解密 · ifEnabled/chatEnabled 功能开关
👤 Users 模块用户
users.service user.entity user-point.entity
实体：User (id/phone/passwordHash/role) · UserPoint (userId/points)
功能：注册赠送100积分 · 手机号待AES加密
📈 Writers 模块作家配额
writers.controller writers.service writer-daily-upload.entity
接口：GET /api/writer/upload-stats
限制：普通用户10章/天 · 管理员无限制
📊 Admin 模块管理
admin.controller admin.service
接口：GET /api/admin/stats
统计：图书数 · 章节数 · 用户数 · IF线数 · 待审核数
🛡️ 公共模块Common
guards/optional-auth.guard decorators/current-user.decorator
crypto.utils doc-parser.utils
OptionalAuthGuard：提取JWT但不强制，未登录也可访问
doc-parser：MD严格模式(#前缀) / TXT宽松模式 章节拆分

七、关键技术细节

安全、性能、工程化要点

🔒 安全
• 密码 bcrypt (cost=10) 哈希存储
• JWT 7天过期 · 前端 localStorage 持久化
• 敏感字段 AES-256-CBC 加密（apiKey / filePath / phone 待实现）
• 路由守卫：token校验 + role校验 (admin路由)
• 下架书隔离：未登录/非作者 不可见
⚡ 性能
• 章节列表滚动分页 (50条/批次)
• axios timeout 10s
• 文件上传串行队列 (避免并发序号错乱)
• marked 客户端 Markdown 渲染
• 阅读设置 localStorage 缓存
🏗️ 工程化
• Vite HMR 开发服务器 · NestJS watch 模式
• TypeORM synchronize:true (开发环境)
• ESLint + Prettier (后端)
• CORS: localhost:5173
• Body parser 500MB 限制
📐 设计模式
• NestJS 模块化 · Controller-Service-Entity
• TypeORM Repository Pattern
• Pinia Composition API Store
• Vue 3 Composition API (script setup)
• OptionalAuthGuard: 认证可选，未登录也能访问
